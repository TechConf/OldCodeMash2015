Configure well known addresses


Configure a well known address for a Coherence cluster. Other members can use this address to enroll in the cluster.

To configure well known addresses:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, expand Environment and select Coherence Clusters.

    The Summary of Coherence Clusters page displays the Coherence cluster configurations that have been created in this domain.
  3. Click the name of the cluster for which you want to configure a well known address.
  4. Select Configuration > Well Known Addresses.
  5. To create a new well known address for this cluster configuration, click New or click on the name of an existing well known address.
  6. Provide or adjust the following cluster configuration settings:
    1. Name: A unique name that identifies this well known address.
    2. Listen Address: The address where this cluster is listening for incoming connections.
    3. Listen Port: The port where this cluster is listening for incoming connections.
  7. Click Save.
  8. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
