Create Jolt connection pools


Oracle recommends that you create one Jolt connection pool for each application running on WebLogic Server.

To create a Jolt connection pool:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the Administration Console, expand Interoperability and select Jolt Connection Pools.
  3. On the Jolt Connection Pools page, click New.

    Note: Once you create a connection pool, you cannot rename it. Instead, you must delete it and create another one that uses the new name.

  4. On the Create a Jolt Connection Pool page, enter a name for the connection, then click Finish.
  5. On the Jolt Connection Pools page, click the name of the connection pool you created.
  6. Click the Addresses tab and provide Jolt Service Listener (JSL) addresses for a connection pool.

    See Configure Jolt addresses.

  7. To secure connections to the Jolt server, Configure a user identity.
  8. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).

After you finish

  • Target Jolt connection pools
  • Configure Jolt startup and shutdown classes
  • Because Jolt requires a server startup class, you must restart the Administration Server after creating a connection pool. You also must restart the Administration Server if you modify an existing connection pool.
