Clone Coherence servers


Cloning servers creates new servers based on existing ones. After cloning, update the configuration of the cloned servers if needed.

Note: The Coherence servers feature is deprecated and remains supported for backwards compatibility. Coherence tiers should be configured to use Managed Coherence Servers instead. See Configure and deploy Coherence deployment tiers

To clone Coherence servers:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, expand Environment and select Coherence Servers.

    The Coherence servers table displays all the Coherence servers defined in the domain.
  3. Select the check box next to the name of the server you want to clone and click Clone.
  4. On the Coherence Server Properties page, adjust the server properties values as needed.

    For information about these fields, refer to Configuration Options.

  5. When you are finished making changes, click Finish.

    The new server appears in the Summary of Coherence Servers table.

  6. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).

After you finish

(Optional) Do any of the following:

  • Start Coherence servers from the Administration Console
  • Delete Coherence servers
