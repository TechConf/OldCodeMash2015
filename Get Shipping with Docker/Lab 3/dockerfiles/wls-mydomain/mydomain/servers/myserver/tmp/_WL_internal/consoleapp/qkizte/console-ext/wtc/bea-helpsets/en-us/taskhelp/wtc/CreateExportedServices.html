Create exported services


Exported services provide information about how to provide Java application services to remote Tuxedo application environments. To create an exported service:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the Administration Console, expand Interoperability and select WTC Servers.
  3. On the WTC Servers page, select the name of the WTC server on which you want to create exported services.
  4. Select Configuration > Exported.
  5. On the WTC Exported Services page, click New.
  6. In Resource Name, enter a name for your exported service.
  7. In Local Access Point, enter the name of the local access point that exports this service.
  8. In EJB Name, enter the complete name of the EJB home interface to use when invoking this service.
  9. In Remote Name, enter the remote name of the service.
  10. Click OK.
  11. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
