Create imported services


Imported services provide information on how to access services that are available on remote Tuxedo domains. To create an imported service:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the Administration Console, expand Interoperability and select WTC Servers.
  3. On the WTC Servers page, select the name of the WTC server on which you want to create imported services.
  4. Select Configuration > Imported.
  5. On the WTC Imported Services page, click New.
  6. In Resource Name, enter the name used to identify this imported service.
  7. In Local Access Point, enter the name of the local access point that offers this service.
  8. In Remote Access Point List, enter a comma-separated failover list that identifies the remote domain access points through which resources are imported.
  9. In Remote Name, enter the remote name of the service.
  10. Click OK.
  11. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
