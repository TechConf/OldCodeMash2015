Configure a user identity


You can secure access to your Jolt connection pool by specifying a user name and password. Applications must provide this user name and password to access the connection pool.

If Tuxedo requires clients to provide user credentials, you must specify the user name and password of an existing Tuxedo user. You must also enable the connection pool to pass the security context to Tuxedo.

To configure a user identity:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the Administration Console, expand Interoperability and select Jolt Connection Pools.
  3. On the Jolt Connection Pools page, click the name of a connection pool.
  4. Click the Configuration: User tab.
  5. On the User tab, in User Name, enter a user name.
  6. In User Password, enter the user's password.
  7. If the Tuxedo authentication level is set to USER_AUTH or APP_PW:
    1. In User Role, enter a Tuxedo user role.
    2. In Application Password, enter a Tuxedo application password.
  8. Click Save.
  9. If Tuxedo requires clients to provide user credentials:
    1. Click the Configuration: General tab.
    2. Check Security Context Enabled.
    3. Click Save.
  10. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
