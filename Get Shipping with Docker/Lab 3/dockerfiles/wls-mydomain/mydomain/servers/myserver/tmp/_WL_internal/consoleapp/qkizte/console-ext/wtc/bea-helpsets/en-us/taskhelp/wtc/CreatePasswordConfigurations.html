Create password configurations


Password configurations provide passwords for inter-domain authentication through access points. To create a password configuration:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the Administration Console, expand Interoperability and select WTC Servers.
  3. On the WTC Servers page, select the name of the WTC server on which you want to create password configurations.
  4. Select Configuration > Passwords.
  5. On the WTC Password Configurations page, click New.
  6. In Local Access Point, select the name of the local access point to which this password applies.
  7. In Remote Access Point, select the name of the remote access point to which this password applies.
  8. In Local Password, enter the local password for this configuration.
  9. In Local Password IV, enter the initialization vector used to encrypt the local password.
  10. In Remote Password, enter the remote password for this configuration.
  11. In Remote Password IV, enter the initialization vector used to encrypt the remote password.
  12. Click OK.
  13. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
