Create Tuxedo queuing bridge redirections


Redirections are used to configure a one-to-one connection between the JMS interface and Tuxedo application environment. To create a redirection:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the Administration Console, expand Interoperability and select WTC Servers.
  3. On the WTC Servers page, select the name of the WTC server on which you want to create Tuxedo queuing bridge redirections.
  4. Select Configuration > Redirections.
  5. On the WTC Redirections page, click New.
  6. Modify the redirect attributes for your environment.

    See Configuration Options.

  7. Click OK.
  8. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
