Configure Coherence Grid Archives (GARs)


To configure Coherence Grid Archives (GARs):

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the console, select Deployments.

    The Deployments table in the right pane lists the deployments currently installed on WebLogic Server. The Type column specifies the type of deployment.
  3. In the Deployments table, select the Coherence Grid Archive (GAR) you want to configure.
  4. Select Overview to view general information about the Coherence Grid Archive (GAR), such as its name, path to the application source files, and deployment plan. You can also edit some of the information on this page, such as deployment order. Click Save to save any changes.
  5. Select Configuration to view configuration information for the Coherence Grid Archive (GAR).
  6. Select Targets to specify the WebLogic Server instances and clusters on which you would like to deploy the Coherence Grid Archive (GAR). Click Save to save any changes.
  7. Select Control to start, restart, or stop the Coherence Grid Archive (GAR).
  8. Select Notes to include any additional information that describes the configuration of this Coherence Grid Archive (GAR). Click Save to save any changes.
  9. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
