Create WTC servers


The WebLogic Tuxedo Connector (WTC) provides interoperability between WebLogic Server applications and Tuxedo services. It allows WebLogic Tuxedo Connector clients to invoke Tuxedo services and Tuxedo clients to invoke WebLogic Server applications, such as EJBs and servlets. To configure WebLogic Tuxedo Connector, you must create a WTC server to establish a session connection to link WebLogic Server to Tuxedo. To create a WTC server:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the Administration Console, expand Interoperability and select WTC Servers.
  3. On the WTC Servers page, click New.
  4. On the Create a New WTC Server page, enter the name of the WTC server.
  5. Click OK.
  6. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).

After you finish

  1. Configure security for local access points.
  2. Create password configurations.
  3. Target WTC servers.
